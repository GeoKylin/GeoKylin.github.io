“我想最大程度地减少编程的难度，因此想最大程度地减少编程的劳动。这就是我设计 Ruby 的主要目标。我想让自己快乐地编程。”——松本行弘（Matz），Ruby之父

Twitter 创意总监罗宾·斯隆（Robin Sloan）在他的长篇小说《生命之书》中将红宝石（Ruby）语言赞誉为最为优美的编程语言，Ruby 语言的优美正符合了 Matz 在设计 Ruby 时的指导思想——“Ruby 是作为一门令程序员快乐的语言而设计的”。现在，让我们来了解一下这一饱受赞誉的编程语言 Ruby。

Ruby 的发展历史

Ruby 被许多人认为是一种“新”编程语言，但实际上是由松本行弘“Matz”（Yukihiro Matsumoto）于 1993 年开始着手研发，并于 1995 年公开发布的。Matz 自我描述为一个“语言极客”，是 Perl 的粉丝，他对 Ruby 的想法是创建一种像 Perl 这样灵活而强大的语言，但在其语法上更具表现力，甚至可以具有类似于英语那样的可读性。

1995 年 12 月 21 日，日本国内新闻组第一次公开发布了 Ruby 0.95，随后，Ruby 凭借其独特的魅力在日本迅速成长，但直到 2000 年，仍然没有除了日文外的任何 Ruby 文档，同年，Ruby 在日本比 Python 更受欢迎。

2000 年，敏捷编程的先驱 Dave Thomas 被 Ruby 迷住，并决定创建 Ruby 的其它语言文档。自此，Ruby 在英语世界中缓慢成长，到 2005 年，美国的 Ruby 社区已达数百个。

其后，随着 Rails 的大紫大红之后，Ruby 已经逐渐成为主流脚本语言中的重要一员，并在各种平台上开花结果。如今，Ruby 经常位于全球编程语言成长和流行度指数的前十名（比如 TIOBE），各种 Ruby 及 Rails 方面的图书也如雨后春笋般涌现。

Ruby 是一门完全面向对象的编程语言

Ruby 是一门动态的、解释性的、反射的、面向对象的通用编程语言。

在 Ruby 中，一切皆对象，是一 门完全的 面向对象的编程语言。所有的信息和代码都拥有属性和行为，即便是简单的数值字面量，以及 true、false 和 nil（Ruby 版的 null）也是如此。

在许多语言中，数字和其它基本类型都不是对象。而 Ruby 受 Smalltalk 语言影响，所有类型都可以赋予方法和实例变量。这一点让 Ruby 特别易于使用，因为适用于对象的规则适用于 Ruby 中的一切。

Ruby 语言的特性

  • 字符串是可变的：Ruby 定义了各种方法对字符串进行就地更改，包括插入、替换、删除、追加等，而不改变对象本身，这与 Java 有着很大的不同；

  • Ruby 条件式和循环中：nil 值被当做 false 来处理，而任何其他值都和 true 一样处理，也即期望空字符串 "" 能像 false 那样工作，这与 C 或者 JavaScript 有着很大不同，因为它们期望 0 能够像 false 那样工作；

  • 变量没有类型：Ruby 的变量可以保存任何类型的数据；

  • Ruby 语言很优雅：可以做到不需要任何注释就可以读懂；

  • Ruby 是一门相当灵活的语言，允许用户改变自身：Ruby 的核心部分可以更改，也可以重新定义。Ruby 的运算符只是语法糖（syntactic sugar），随时可以重定义；

  • Ruby 提供不依赖操作系统的线程：所以，不管操作系统是否支持，Ruby 支持的所有操作系统都可以使用多线程，甚至是 MS-DOS；

  • Ruby 高度可移植：它主要在 GNU/Linux 上开发，但也支持很多类型的 UNIX，以及 macOS，Windows，DOS，BeOS，OS/2 等；

  • Ruby 完全免费，还可以自由使用、复制、修改和分发。

Ruby 中的一些概念

  1. 迭代器与代码块：times 和 upto 是由整数对象实现的方法，它们被称为迭代器，包含在花括号中的代码被称为代码块，数组和哈希对象也有相似的方法；

  2. 方法：Ruby 中用 def 来定义方法，以 end 结尾，方法中最后一个表达式的值是方法的输出；

  3. 赋值：Ruby 支持并行赋值，允许赋值表达式左右包含多个值或变量；

  4. 前缀和后缀：=结尾的方法与赋值操作的功能类似，?结尾的方法用于返回 Boolean 类型的值，!结尾的方法会修改原对象。全局变量以$为前缀，实例变量以@为前缀，类变量以@@为前缀。
